================================================================================
                           OVERVIEW
================================================================================

The Nexus Complaint Tracking System is an enterprise-grade, full-stack web
application for managing service complaints and maintenance requests. It supports
three distinct user roles (Admin, Worker, User) and is packed with AI-driven
automation, real-time monitoring, self-service tools, and professional analytics.

It has been developed in three progressive tiers:
  Tier 1 - Foundation       : Core authentication, ticket management, category routing
  Tier 2 - Enterprise Prime : AI automation, audit logging, worker portal, SLA management
  Tier 3 - Infinity Beyond  : Predictive analytics, self-service wiki, global search

================================================================================
                           USER ROLES
================================================================================

1. USER (Regular)
   - Register and log in
   - Submit new service complaints with image evidence
   - Track ticket progress in real-time
   - View SLA countdowns on each ticket
   - Rate and leave feedback on resolved tickets
   - Add comments to discussions
   - Export individual ticket as PDF

2. WORKER
   - Dedicated "Worker Portal" dashboard
   - View all assigned tasks
   - Move tickets from "Assigned" → "Resolved"
   - Receive email notifications on new assignments

3. ADMIN
   - Full access to the Nexus Analytics command center
   - Manage all complaints: assign workers, update statuses
   - View the Live Activity Feed (real-time audit log)
   - Export analytics data as CSV
   - Use Command-K (OmniSearch) to find any ticket or worker instantly
   - Access the Nexus Vision demand forecast chart
   - Manage the Knowledge Base (add Wiki solutions)

  • On ticket creation, system automatically selects the

  • Fire-and-forget background task — never blocks the response
  • Integrated with audit logging for full traceability
  • POST logic in AssignmentService

[AUDIT LOGGING]
  • AuditLog model tracks: TICKET_CREATED, STATUS_UPDATE,
    WORKER_ASSIGNED, AUTO_ASSIGNMENT events
  • Immutable record with user, timestamp, and details

[CRISIS MODE]
  • Admin Dashboard detects active "Emergency" priority tickets
  • Entire interface shifts to red emergency theme
  • Page title changes to "System Hub: CRISIS MODE"

  • Real-time Live Activity Feed on the Admin Dashboard
  • Polls /api/admin/logs every 5 seconds
  • Terminal-style animated log entries

  • Every ticket shows a live HH:MM:SS countdown to SLA deadline
  • SLA deadlines auto-calculated on creation by priority:
      Emergency → 4 hours
      High      → 24 hours
      Medium    → 3 days
      Low       → 7 days
  • Color shifts: Green → Yellow (due soon) → Red (overdue)

[WORKER PORTAL]
  • Dedicated dashboard for workers at /worker
  • Shows all non-resolved assigned tickets
  • Workers can press "Start Work" or "Mark Resolved" inline
  • Role-based routing in Navbar

[PDF EXPORT]
  • One-click PDF report generation on any ticket detail page
  • Includes ticket metadata, description, and comment history
  • Powered by jsPDF + jspdf-autotable

[CSV EXPORT]
  • Admin can export category and priority analytics as CSV
  • Instant download from the Admin Dashboard

[PERFORMANCE LEADERBOARDS]
  • "Field Elite" leaderboard showing top workers on Admin Dashboard
  • Efficiency score badges (PRO LEVEL, RATING)
  • Workers ordered by resolution count and score

[NEXUS WIKI — SELF-SERVICE DEFLECTION]
  • Knowledge base of "Quick Fix" guides seeded in the database
  • While typing a complaint, the deflection engine checks keywords
    against the Solution library (GET /api/wiki/deflect)
  • If a match is found, a collapsible "Nexus Wiki — Quick Fix"
    panel appears inline with step-by-step solution
  • Reduces ticket volume by empowering users to self-resolve
  • Admins can add new solutions via POST /api/wiki

[NEXUS VISION — PREDICTIVE DEMAND ANALYTICS]
  • 7-day demand forecast chart on the Admin Dashboard
  • Uses 3-period moving average of historical complaint trends
  • Custom animated bar chart (no extra charting dependency)
  • Today's bar is highlighted for orientation

[COMMAND-K OMNISEARCH]
  • Global keyboard shortcut: Ctrl+K (or Cmd+K on Mac)
  • Opens a blazing-fast full-screen search overlay from any page
  • Searches tickets (title, description) and workers (name, email) in parallel
  • Keyboard navigation: ↑↓ to browse, ↵ to open, Esc to close
  • Debounced API call to GET /api/admin/search?q=...
  • Admin-only (search button also visible in Navbar)

================================================================================
                         HOW TO RUN THE PROJECT
================================================================================

PREREQUISITES:
  - Node.js v18+
  - npm

SETUP:
  1. cd "complaint tracking system"
  2. npm install
  3. Create a .env file with:
       DATABASE_URL="file:./complaints.db"
       JWT_SECRET="your-secret-key"
       PORT=5000
  4. npx prisma db push     (creates and migrates the SQLite database)
  5. npm run dev            (starts both frontend and backend concurrently)

ACCESS:
  - Frontend : http://localhost:5173
  - Backend  : http://localhost:5000

DEFAULT ADMIN:
  - Register a user then manually set isAdmin=true in the database,
    or use the seed script if available.

================================================================================
                    RESOLUTION VALIDATION & ACCOUNTABILITY
================================================================================

Our system ensures that complaints cannot be falsely marked as resolved through 
multiple validation layers:

1. Proof-of-Fix Mechanism:
   The assigned worker must upload valid evidence (images, location data, 
   timestamps) before marking a task as completed. The resolution option remains 
   disabled without this proof.

2. User Confirmation System:
   Once an issue is marked as resolved, the user receives a notification and 
   must either accept or reject the resolution. The complaint is only closed 
   after user approval.

3. Reopen Feature:
   Users can reopen complaints if they are not satisfied with the resolution.

4. Audit Logs:
   All activities are recorded, ensuring full transparency. Every action taken 
   by admins and workers is tracked with timestamps.

5. SLA Tracking:
   Monitors resolution times. Unusually fast or suspicious resolutions are 
   automatically flagged for admin review.

6. AI-Based Validation:
   The system can incorporate AI-based validation, where before-and-after 
   images are analyzed to detect improper or fake resolutions.

Overall, the system is designed to maintain transparency, prevent misuse, and 
ensure accountability at every stage of the complaint resolution process.
